# アップデートログ (2026/02/07)

今回のセッションで実施した主要な修正および機能改善の内容です。

## 1. 座標系と配置の改善
- **軸範囲変更時の挙動修正**: 軸の最小値・最大値を変更した際、アイコンの数値（座標データ）が勝手に書き換わらないように修正しました。
- **クランプ処理の解除**: 範囲を狭めた際にアイコンが枠の端に強制移動されないようにし、データを保護しました。
- **アイコンの潰れバグ修正**: 右端や密集地でアイコンが横に潰れる問題に対し、`flex-shrink-0`、`min-width/height`、`whitespace-nowrap` を適用して正円を維持するようにしました。

## 2. 操作性 (UX) の向上
- **スクロール挙動の最適化**: 
  - プロットエリア内でのスワイプによる画面スクロールを可能にしました。
  - アイコン操作時のみスクロールをロックするよう制御し、誤操作を防止しました。
- **モバイル対応**:
  - プロジェクト一覧のリネーム・削除ボタンを、タッチデバイスでは常時表示するように変更しました。
  - アイコンに `touch-none` を設定し、ドラッグ時のブラウザ標準挙動との干渉を解消しました。

## 3. デザインとレイアウト
- **タイトル配置の変更**: タイトルをプロット図の中央から上部（外側）へ移動し、アイコンとの重なりを解消しました。
- **UIの整理**: 画面下部のTips（💡）枠を削除し、メインコンテンツを広く使えるようにしました。

## 4. 画像保存機能の安定化
- **選択状態の自動非表示**: 画像保存時のみ、アイコンの選択枠や拡大エフェクトを一時的に消すことで、綺麗な画像を保存できるようにしました。
- **Safari (iOS) 対応**: 
  - iPhone等で画像が乱れたり影が出たりする問題に対し、キャプチャサイズの明示化とUIシャドウの除外を行いました。
  - レンダリング待機時間を調整し、重なりの多い画像でも安定して生成できるようにしました。
- **出力解像度の最適化**: メモリ負荷を考慮し、高画質を維持しつつ安定する `pixelRatio: 2` に調整しました。
